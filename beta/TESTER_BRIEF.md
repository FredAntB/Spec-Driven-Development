# Beta Tester Brief — spec-driven-development v1.0-beta

Thank you for testing. This takes about 30–60 minutes and your
natural, uncoached experience is the entire point.

---

## The one rule

**Use your own words.** Do not try to phrase things "correctly"
to trigger the skill. Say what you would actually say to a
colleague. If the skill doesn't understand you, that is a bug,
not user error — and it's exactly the data we need.

---

## What you're testing

A Claude skill that helps developers write specs before they write
code, and keeps multiple AI coding tools consistent with each other.

When you open a new conversation with Claude (with the skill active),
just describe what you want to do. The skill handles the rest.

## Installation

**Option A — Claude.ai (Chat tab)**
The skill is installed in your account. Start a new conversation
and begin with your scenario description.

**Option B — Claude Code (Code tab)**
1. Download the repo as a zip from GitHub
2. Extract it to a folder on your machine
3. In the Code tab: Select Local → choose that folder
4. The CLAUDE.md file bootstraps the skill automatically

---

## How to file feedback

Go to: github.com/[your-username]/spec-driven-development/issues

Use the appropriate template:

- **Bug** — the skill did something wrong or generated bad output
- **Miss** — the skill didn't activate when you expected it to
- **Friction** — the skill worked but something felt off or confusing

One issue per problem. Include the exact phrase you used — that
is the most important piece of information.

---

## What we are NOT asking you to do

- You don't need to read SKILL.md or any reference files
- You don't need to know what "SDD" is before starting
- You don't need to give positive feedback — honest is better
- You don't need to finish if the skill completely fails you
  (filing a "abandoned because X" issue is more valuable than
  pushing through a broken experience)

---

## At the end

File one final issue titled "Beta wrap-up — [your tester number]"
with:
- Scenario: [your scenario]
- Completed: yes / partial / no
- Satisfaction: 1–5
- Most useful thing the skill did
- Most frustrating thing
- One thing you'd change
